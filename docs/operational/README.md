# Operational 分类

> 运维、资源与服务状态相关能力

## 定位

Operational 分类提供**运维、资源与服务状态**相关能力，帮助您查询 API 配额、余量、服务状态等运维辅助信息，适用于检查额度、监控资源可用性、管理账户资源等场景。

## 支持的鉴权方式

| 鉴权方式 | 支持 | 说明 |
| --- | --- | --- |
| **API Key** | ✅ | 适用于查询 API 配额、余量、服务状态等 |

## 能力范围

- **查询余量 API**：查询当前 API 调用余量
- **查询配额**：查询 API 调用配额与使用情况（API Key）
- **查询服务状态**：查询 ZAN 服务可用性状态
- **运维辅助信息**：获取运维相关的配置与状态信息

## 典型场景

- **检查 API 配额剩余**：在调用前或定期检查 API 配额是否充足
- **监控资源可用性**：了解服务状态，确保调用可用

## API 列表

| API 名称 | 描述 | 鉴权方式 | 文档链接 |
| --- | --- | --- | --- |
| `query-quota` | 查询当前账户的 API 调用配额与使用情况 | API Key | [query-quota.md](./query-quota.md) |

## Agent 使用建议

当用户关注以下关键词时，**优先使用 Operational 分类**：

- **额度、余量、配额**：用户想了解还能调用多少次、剩余多少额度
- **状态、可用性**：用户想确认服务是否正常、资源是否可用

**推荐调用时机**：

- 用户明确询问「还有多少额度」「配额用完了吗」
- 在批量调用或 Agent 自动化任务前，可主动查询配额，避免调用失败

若用户需求偏向「链上查询」「数据分析」「交易加速」，请参考对应分类：[Chain RPC](../chain-rpc/README.md)、[Data](../data/README.md)、[Trading](../trading/README.md)。
